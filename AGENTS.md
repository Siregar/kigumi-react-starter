# kigumi-react

React starter project using Kigumi CLI and Web Awesome web components.

## Tech Stack

- React 19, TypeScript, Vite
- Web Awesome (web components with built-in a11y)
- Kigumi CLI for component generation and theming

## Project Structure

- `src/components/ui/` -- **Generated by Kigumi CLI. Do not edit manually.**
- `src/components/examples/` -- Example screens (LoginExample)
- `src/lib/kigumi.ts` -- **Generated.** Web Awesome setup and theme classes
- `src/styles/layers.css` -- **Generated.** CSS cascade layer definitions
- `src/styles/theme.css` -- **Generated.** Custom theme overrides (editable via `npx kigumi theme`)
- `src/App.tsx` -- Application root
- `src/main.tsx` -- Entry point

## Key Rules

1. **Never edit generated files directly.** Use Kigumi CLI commands instead:
   - `npx kigumi add <component>` -- add a component
   - `npx kigumi update` -- update components to latest templates
   - `npx kigumi theme <name>` -- change theme
   - `npx kigumi brand <color>` -- change brand color
2. **`class` not `className`** on `<wa-*>` elements -- web components use the native `class` attribute.
3. **CSS cascade layers** (base < theme) handle specificity. Never use `!important`.
4. **`--wa-*` CSS custom properties** are the token layer for theming.

## Component Pattern (React)

Wrappers use `forwardRef` + `useImperativeHandle` for ref methods, `useEffect` for event listener cleanup. Props are typed interfaces matching Web Awesome attributes.
